# CLAUDE.md — проект Bosnia and Herzegovina Guidebook

Контентный сайт-справочник по всей Боснии и Герцеговине **только на английском языке (en)** — единственный язык, на корне `/` (ru/uk НЕ заводятся, EN-only навсегда): достопримечательности, города, еда (директория проверенных мест), маршруты, транспорт, аренда авто, страховки, развлечения, релокация, новости. Места отбираются по высокому рейтингу (>4★ из нескольких источников) и сверяются с опытными путешественниками, гидами и местными жителями. Диверсифицированная партнёрская монетизация (аренда авто, трансферы, жильё, страховки, eSIM). Единственный источник правды по продукту, нише и приоритетам — **SPEC.md**. Этот файл — правила работы внутри репозитория. При конфликте приоритет: SPEC.md → CLAUDE.md → всё остальное.

Сайт — форк общего движка сети путеводителей (донор-форк снят, контент-модель переопределена под Боснию). Общие стандарты, гейты и дисциплина — на уровне хаба (`docs/STANDARDS.md`, `docs/NEW-SITE-BLUEPRINT.md`). Здесь — специфика Боснии.

## Стек и принципы
- **Astro 6 + Content Collections**, полная статика (SSG). Никаких SSR, БД, CMS.
- **Tailwind CSS 4**. Дизайн-токены — только из `src/styles/tokens.css` (SPEC §10). Хардкод hex-цветов в компонентах запрещён.
- **Без React.** Интерактив — нативный JS внутри Astro-компонентов, и только там, где без него нельзя: карта Leaflet, мобильное меню, лайтбокс.
- Хостинг: **Cloudflare Workers (Static Assets)**. Деплой = push в `main` (Workers Build: `npm run build` → `wrangler deploy`). Превью-деплои на ветках.
- Изображения: webp/avif через `astro:assets`, lazy-load, каждая ≤ 200 КБ.

## Гео-модель Боснии (синхронно: content.config.ts ↔ i18n ↔ check-enums)
- **Регионы (`REGIONS`) — 3 турлогических макрорегиона** (не 142 общины): `herzegovina` (юг/средиземноморье: Mostar, Blagaj, Počitelj, Kravice, Trebinje), `sarajevo-region` (Сараево + олимпийские горы Jahorina/Bjelašnica, Konjic), `northern-bosnia` (Una NP/Bihać, Jajce/Pliva, Sutjeska, Banja Luka, Travnik).
- **Опорные города** (= статьи `category: 'cities'`): Sarajevo (столица, ≥15 фото), Mostar (≥10), Trebinje, Jajce, Bihać, опц. Banja Luka/Konjic.
- **Города директории еды (`EDA_CITY_PAGES`):** sarajevo, mostar, trebinje.
- **Точки «как доехать» (`accessFrom`):** sarajevo, mostar, dubrovnik (хорватский хаб day-trip-трафика).
- **Валюта:** опорная в модели — €; в тексте честно — нац. валюта **BAM / KM (конвертируемая марка)**, евро принимают НЕ везде.

## Железные правила
1. **Один язык — английский (en).** Сайт одноязычный навсегда. Контент лежит только в `src/content/<коллекция>/en/`. `check-parity` следит, чтобы контента вне `/en/` не было. ru/uk НЕ заводить.
2. **Партнёрские ссылки** — только через `/go/{partner}?c={slug}` с `rel="sponsored nofollow noopener"`. Прямые URL партнёрок в контенте запрещены. На статью 1–3 AffiliateBox, не больше.
3. **URL не меняются** после индексации. Ошибся в slug — 301 в `public/_redirects`, а не переименование.
4. **Факты не выдумывать.** Цены, адреса, телефоны, условия — только из брифа `briefs/{slug}.md`. Нет данных → `<!-- TODO: факт от владельца -->` и сказать об этом. Рейтинги/отзывы из внешних источников дословно не перепечатывать (ToS) — использовать как критерий отбора + ссылку. Даты/время/числа — только из источника **текущего года**.
5. **Перф-бюджет (мобайл):** LCP < 2.0 с, CLS < 0.1, JS на странице < 50 КБ. Любая новая зависимость тяжелее 10 КБ — спросить владельца.
6. **Доступность:** контраст ≥ 4.5:1, видимый focus, alt у каждого изображения, рабочая навигация с клавиатуры.
7. **SEO-обвязка каждой страницы:** title ≤ 60 зн., description ≤ 155 (в символах, не байтах), canonical, self-hreflang `en` + `x-default` → тот же URL (сайт одноязычный), OG-картинка, schema.org по типу шаблона (SPEC §14).
8. **Сторонние скрипты:** только одна аналитика — GA4 (SPEC §17). Никаких виджетов соцсетей, онлайн-чатов, тег-менеджеров. Consent Mode v2 (CookieConsent + ga-init denied-by-default) обязателен.
9. **Доки не отстают от кода + commit/push.** После КАЖДОЙ доработки синхронизировать документацию (PROGRESS/HANDOFF/AUDIT, KALENDAR + hub-копия, роадмап) и сразу `git commit` + `git push` в `main` — без напоминаний. Перед push — `git pull --rebase`.

## Контент-конвейер (SPEC §12)
`briefs/{slug}.md` (ключевой запрос, вопросы читателя, ФАКТЫ владельца, цены, фото) → **скелет:** `npm run new -- <тип> <slug> --title "…"` (генерит ОДНУ en-версию + папку фото + DoD-чеклист) → черновик en по шаблону → фото (CC/свои) → **fact-check → content-editor** → коммит, draft снимается.

## Стандарты публикации (объём и фото) — обязательны
**Объём текста:**
- **Статья:** **1200–2000 слов.** Первый абзац — прямой ответ за 2–3 предложения; H2/H3 под вопросы читателя; без «воды» и без FAQ-блока.
- **Новость:** **1500–2000 знаков** (~250–350 слов), **3–5 разделов**, **≥4 внутренних ссылки**.

**Минимум фото** (каждое уникально на материал, **webp ≤200 КБ**, своё или CC/PD — Wikimedia/Openverse/Unsplash — с атрибуцией «автор + лицензия»):
- **Статья:** **≥5** (cover + ≥4 инлайн/галерея).
- **Новость:** **≥2** (cover + ≥1 инлайн).
- **Маршрут:** cover + фото **каждой** остановки.
- **Город:** **≥10**; **столица (Sarajevo):** **≥15**.

**Модель «город = статья».** Город-путеводитель — это статья с `category: 'cities'`; отдельная коллекция `cities` для городов-путеводителей по этой модели НЕ используется.

**Тон.** Живой, человечный, как пишет носитель английского для Tier-1/Western аудитории. **НЕ ИИ-генерёнка** — жёсткий гейт публикации наравне с объёмом и фото. Факты — только из источника/брифа. Валюта — **конвертируемая марка (BAM / KM)**; евро принимают не везде — честно это отмечать.

**Партнёрские ссылки** — только через `/go/{partner}` с `rel="sponsored nofollow noopener"` (правило 2).

**Уникальность намерения (④ гейт сети).** Одна интент-страница: без mirror-дублей («A→B» = «B→A» = одна двунаправленная стр.) и тонкого variant-слайсинга. Гейт `check-dedup` в `npm test` / `npm run qa`. Полные правила — хаб `docs/STANDARDS.md`.

## Перелинковка
Обязательна: статья ≥2 внутренние ссылки, новость ≥4, в обе стороны, без сирот. Кросс-сайт — контекстно обычной markdown-ссылкой на полный `https://`-URL сестринского сайта сети, только когда цель реально резолвится (напр. day-trip Dubrovnik→Mostar → страница Дубровника на Croatia Guidebook). Движок (`rehypeExternalLinks`) сам навешивает `rel="nofollow noopener noreferrer"`.

## Монетизация Боснии (SPEC §16)
- **Ядро дохода:** `localrent` (аренда авто — петля Sarajevo→Mostar→Герцеговина), `kiwitaxi` (трансферы, вкл. Dubrovnik→Mostar), `trip-hotels` (отели), `yesim` (eSIM), `safetywing` (страховка), `aviasales` (рейсы).
- **🔴 ТУР-ПАРТНЁРА НЕТ (Вариант A владельца):** Trip.com `trip-tours` по Боснии = 0 туров; GetYourGuide/Viator НЕ заводим. Day-trip/тур-темы монетизируем авто+трансфер+отель. `trip-tours` оставлен валидным редиректом, но в контент-метках не используется.
- **EKTA НЕ подключена** (в сети сломанный прямой URL без атрибуции); страховку ведём на `safetywing`.

- Не генерировать массово страницы без уникальной фактуры: «SEO-генерёнка» = риск бана и прямое нарушение SPEC §15.
- Не трогать логику `/go/` без обновления таблицы партнёрок в SPEC §16.
- Не заводить GetYourGuide/Viator/тур-партнёра без явного решения владельца (Вариант B).
