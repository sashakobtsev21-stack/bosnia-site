# /news — дайджест новостей (команда → превью → «ок» → публикация)

Цель: по одной команде принести владельцу **готовый отобранный дайджест** свежих
тревел-новостей Боснии и Герцеговины и опубликовать **только то, что он одобрил**. Никакой слепой
автопубликации. Источник правды по продукту — SPEC.md; правила — CLAUDE.md.

**Жёсткие принципы (нарушение = бан/искажения):**
- **Правило 4 — факты не выдумывать.** Каждая новость = пересказ РЕАЛЬНОГО материала с источником в `sources`. Не подтверждено источником → не публикуем (или `<!-- TODO -->`). Никаких выдуманных цифр/дат/цен.
- **🔴 ДАТЫ И ВРЕМЯ — сверять по источнику ТЕКУЩЕГО года (критично!).** Любую дату/время/число бери только из материала, чья СОБСТВЕННАЯ дата публикации = текущий год. **Реальные грабли конвейера:** дата ежегодного фестиваля была взята из прошлогодней статьи → опубликовали прошлогодние числа вместо реальных дат этого года; владелец справедливо назвал это фейком. Для **ежегодных событий** (например, летние фестивали в Сараево/Мостаре/Требине, сезонные открытия) НИКОГДА не копируй даты из прошлогодней статьи — найди подтверждение дат именно этого года в свежем источнике (новость с датой публикации этого года / официальный портал организатора). Если дату текущего года подтвердить нечем — НЕ публиковать дату (или новость).
- **§15 копирайт/ToS.** Чужой текст НЕ копировать дословно — только свой пересказ + ссылка на первоисточник. Заголовки тоже свои.
- **Фото обязательно** у каждой новости (cover ≤200КБ webp, легальное: Unsplash/Commons с атрибуцией).
- **Только английский (en).** Bosnia and Herzegovina Guidebook — одноязычный EN-first сайт. Новость создаётся ТОЛЬКО в `src/content/articles/en/<slug>.md`. Никаких ru/uk-версий и переводчиков.
- **НЕ публиковать без «ок» владельца.** До одобрения всё лежит `draft:true`.

## Аргументы
`/news [N] [окно]` — `N` = сколько новостей в дайджесте (по умолчанию **топ 3–5**),
`окно` = период (по умолчанию **последние 24 ч**). Без аргументов — дефолты.

## Тематика (= фильтр мусора). Берём ТОЛЬКО тревел-релевантное
**Беру** (влияет на поездку туриста):
- въезд / визы / границы / пограничные правила (учитывай хук «US — длительное пребывание без визы»);
- авиа: рейсы, новые направления, отмены, аэропорты (**Сараево — SJJ**, **Тузла — TZL**, **Мостар — OMO**, **Баня-Лука — BNX**);
- цены / валюта / **курс конвертируемой марки (BAM/KM)** / налоги и сборы для туриста;
- погода / стихия / ЧС / безопасность / закрытия дорог и перевалов;
- открытия и закрытия мест, отелей, транспорта, паромов, музеев;
- крупные события, фестивали, праздники с датами.

**Выкидываю:** внутренняя политика, выборы, криминал, спорт, светская хроника,
бизнес-сводки без связи с туризмом, реклама, кликбейт, «топ-10»-листиклы, гороскопы.

## Источники (правит только владелец)
Издания (EN): **Sarajevo Times, N1 Bosnia, Balkan Insight (Bosnia), bne IntelliNews, Total Croatia News (для day-trip-контекста)**.
**Google News RSS** по запросам: `Bosnia travel`, `Sarajevo airport`, `Bosnia visa`,
`Bosnia border`, `Bosnia and Herzegovina tourism`, `Mostar`.
Официальные: **МИД Боснии и Герцеговины (`mvp.gov.ba`)**, **Погранполиция БиГ (`granpol.gov.ba`)** (визы/въезд),
**Центробанк БиГ (`cbbh.ba`)** (курс марки BAM), **аэропорты** (Сараево SJJ, Тузла TZL, Мостар OMO, Баня-Лука BNX).
_Инструменты сбора: `WebSearch` (с указанием свежести) + `WebFetch` (открыть материал и
прочитать факты). Если источник недоступен — пропусти, не выдумывай._

## Шаги (по порядку, гейты не пропускать)

1. **Сбор.** Пройди по источникам через WebSearch/WebFetch, собери кандидатов за окно
   (по умолч. 24 ч). Для каждого зафиксируй: заголовок, суть, дату, URL первоисточника.

2. **Фильтр мусора — 3 ступени (финальный судья — ты, не регэкспы):**
   - **релевантность** строго по списку «беру» выше — всё остальное отсев;
   - **дедуп/кластеризация** — одну историю из нескольких источников схлопни в один пункт, возьми лучший первоисточник (официальный > крупное издание > агрегатор);
   - **анти-кликбейт/реклама** — выкинь рекламу, пустые и кликбейт-заметки.
   Отбери **топ N** по важности для туриста (свежесть + влияние на поездку).

3. **ПРЕВЬЮ владельцу — СТОП, жди ответа.** Покажи дайджест-таблицу: `№ | тема-тег |
   заголовок (en) | 1–2 строки сути | источник (ссылка) | фото (что поставим)`.
   Кратко перечисли, что отсеял и почему (1 строка). **Ничего пока не пишем в контент.**
   Жди: владелец скажет, какие брать («эти — да», «эту убрать», «добавь…»).

4. **После «ок» — по каждой одобренной новости:**
   - **Скелет:** `npm run new -- news <slug> --title "…"` (slug — латиница-дефис, описательный, стабилен навсегда; при риске коллизии добавь месяц, напр. `-june-2026`). Создаст en-черновик `draft:true` + папку фото.
   - **Рерайт EN-first** своими словами, НАСЫЩЕННО: заголовок ≤60, description ≤155; **3–5 коротких разделов** с КОНКРЕТНЫМИ выверенными фактами (числа, даты, время, расписание, цены-если-есть, **суммы в марках BAM/KM**) + «что это значит для поездки» + раздел с более широким контекстом (смежные события/рейсы/тренды по теме, всё со своим источником). Несколько источников в `sources:` (не один). `publishedAt`=наша дата публикации. **≥4 внутренние ссылки** на наши статьи (гейт `check-interlinks` требует ≥2, но цель — насыщеннее): профильный путеводитель/раздел + соседи кластера (город, транспорт, визы, аренда, «когда ехать» и т.п.). По возможности **1–2 инлайн-figure** с уникальным фото (не дублировать кадры). Партнёрские — только через `/go/{partner}?c=<slug>`, по теме и не натужно. НЕ раздувать в лонгрид — это всё ещё новость, но плотная и полезная, а не «на отвяжись».
   - **Фото:** подбери легальное по теме — `node scripts/commons-candidates.mjs "<тема>"` (или unsplash-кандидаты, если скрипт есть), **отбери глазами** (Read рендерит), собери `build-gallery.mjs` → cover webp ≤200КБ в `public/images/<slug>/` (или общий кадр города). `coverCredit` = автор + лицензия. Нет точного кадра события — жанровый кадр города/темы (Сараево, Мостар, Требине, Яйце, Бихач, Баня-Лука), в `alt`/кредите честно (это иллюстрация, не само событие). **Каждое фото уникально** — не тащить кадр из другой статьи.
   - **Переводов нет:** сайт en-only; ru/uk-файлы не создавай и не публикуй.

5. **Гейт.** `npm run qa` → дождись **GO** (чинит парность/H1≤60/desc≤155/≥2 ссылки/cover≤200КБ). NO-GO — не публикуем, чиним.

6. **Публикация.** Сними `draft:false` у одобренных (остальные кандидаты НЕ коммить — либо удали скелеты, либо оставь `draft:true`). Один коммит на батч → пуш в `main` (правило always-commit-push-main). Cloudflare соберёт; свежесть/ретеншн уже работают (окно раздела 10 дней / главной 2 дня + ежедневный `daily-news-rebuild.yml`).

7. **Синхронизация доков** (правило update-docs-after-each-change): отметь прогон в PROGRESS/HANDOFF при необходимости.

## Запреты
Не публиковать без «ок» владельца. Не копировать чужой текст/заголовки дословно (§15).
Не выдумывать факты (правило 4). Не публиковать новость без фото и без источника.
Не создавать ru/uk-версии. Не тащить политику/криминал/спорт/рекламу.
Прямые партнёрские URL запрещены — только `/go/`.
